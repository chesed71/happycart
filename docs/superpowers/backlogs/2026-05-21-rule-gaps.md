> 시드 큐레이션 / 실제 제품 입력하면서 발견되는 룰 v1.0.0의 한계, false negative,
> false positive 사례를 누적 정리한다. v1.1.0 / v1.2.0 작업 시 일괄 반영.

- 작성일 시작: 2026-05-21
- 관련 스펙: `docs/superpowers/specs/2026-05-20-happycart-clean-eating-design.md` §5
- 룰 패키지: `packages/happycart_rules/lib/src/`

## 0. 룰 v1.0.0 매칭 범위 (요약)

### ✅ 잡는 것 (Not Okay 신호)
- 인공감미료: 아스파탐, 수크랄로스, 아세설팜칼륨, 사카린
- 인공색소: 적색40호, 황색5/6호, 청색1호, 적색3호
- 고과당 옥수수시럽 (HFCS)
- 정제 씨앗유: 대두유, 카놀라유, 옥수수유, 정제 해바라기씨유, 면실유
- 경화유 / 부분경화유 / 트랜스지방
- 합성 보존제: BHA, BHT, TBHQ
- 아질산나트륨, 질산나트륨
- 카라기난
- 유화제: 폴리소르베이트80, DATEM, mono/diglycerides
- 향료 (구체 명시): "natural flavors", "천연향료", "합성착향료", "인공향료"
- 표백/강화 밀가루
- 브롬산칼륨
- 말토덱스트린

### ❌ 의도적 비대상 (현재 v1.0.0)
- 일반 정제 설탕 (sugar)
- 일반 밀가루 (강화/표백 표기 없는 경우)
- 팜유 / 팜스테아린유 / 야자유 (seed oil 아님)
- 라벨에 "향료"만 적힌 케이스 (구체 명시 없는 경우)
- 일반 정제염 / 가공 코코아
- 우유 / 유당 / 레시틴 (단순 표기)

| 항목 | 값 |
|---|---|
| 바코드 | `8801037088168` |
| 브랜드 | 오레오 (동서식품) |
| 용량 | 100g (50g × 2봉지) |
| 기대 verdict | `not_okay` |
| **계산 verdict** | `okay` ⚠️ |
| 매칭된 bad | (없음) |
| 매칭된 good | (없음) |

**라벨 원문 (요지)**:
> 밀가루(밀:미국산,호주산), 설탕, 분당, 팜스테아린유(말레이시아산), 야자유, 가공유지(올리브유), 식물성유지, 해바라기유, 폴리글리세린축합리시놀레인산에스테르, 포도당, 옥수수전분, 팜유(말레이시아산) 67%, 정제소금, 코코아분말 33%, 팽창제, **향료**, 레시틴, 유당, 우유, **바닐린**

**매칭 안 된 이유**:
- "밀가루" → 룰은 "표백/강화 밀가루"만 매칭
- "팜유", "팜스테아린유", "야자유" → seed oil 카테고리 아님
- "해바라기유" → 룰은 "정제 해바라기씨유"만 매칭 (alias 부족)
- "향료" → 룰은 "natural flavors", "천연향료", "합성착향료", "인공향료"만 매칭
- "바닐린" (vanillin) → 합성 바닐라향이지만 룰 키워드 없음
- "폴리글리세린축합리시놀레인산에스테르" (E476) → 유화제이지만 룰 키워드 없음
- "레시틴" → 대두 레시틴 추정이지만 alias 없음

**제안되는 룰 보완**:
1. 카테고리 `palm_oil_processed` 신설: 팜유, 팜스테아린유, 야자유, 경화 팜유 등
2. 카테고리 `vague_flavor`: 단순 "향료" 표기도 잡기 (성분 투명성 낮음)
3. `artificial_flavors` alias 확장: "바닐린", "vanillin"
4. `emulsifier_concern` alias 확장: "폴리글리세린지방산에스테르", "E476", "리시놀레인산에스테르"
5. `seed_oil` alias 확장: "해바라기유" (정제 표기 없어도 가공품에 들어가면 정제로 추정)
6. (논의 필요) 카테고리 `refined_sugar` 또는 `refined_grain`: 일반 설탕/밀가루도 잡을지

### Case 2: 동원 순참치액 (2026-05-21) — 룰 일관성 검증 ✅

| 항목 | 값 |
|---|---|
| 바코드 | `8801047216438` |
| 브랜드 | 동원 (동원F&B) |
| 용량 | 500g |
| 기대 verdict | `okay` |
| **계산 verdict** | `okay` ✅ (일치) |
| 매칭된 bad | (없음) |
| 매칭된 good | (없음) |
| 영양 참고 | 나트륨 100g당 11,090mg (RDI 555%) |

**라벨 원문**:
> 가다랑어추출물 80% (혼합간장, 정제수, 정백당, 정제소금, 참치엑기스, 참치농축액), 참치엑기스 33%, 멸치엑기스 21.54%, 한식간장 4.8% (정제염, 정제소금, 표고버섯추출물), 정백당, 정제염, 양조식초, 표고버섯추출물, 토마토페이스트

**관찰**:
- 자연 추출물 베이스(가다랑어/멸치/참치/표고버섯) + 한식간장 + 양조식초로 비교적 깔끔한 라벨
- 우리 룰 v1.0.0 의 "명백한 합성 첨가물" 카테고리에 매칭되는 키워드가 없음
- 합성 보존제·인공감미료·인공색소·HFCS·seed oil·경화유 모두 ❌
- **ingredient 룰 v1.0.0 의 의도된 결과 — 일관성 검증 OK**

**룰의 의도된 한계 (보완 검토 필요)**:
- **영양 임계 미반영**: 본 제품은 나트륨 100g당 11,090mg (RDI 555%) 로 매우 짠 조미료. clean-eating 일반 견해는 "정제염 다량 = 주의"인데 우리 룰은 ingredient-only.
- **"정백당", "정제염" 다량**: 자연 추출물 사이에 정제 설탕·소금이 끼어있지만 v1.0.0은 매칭 X
- **"엑기스" / "추출물" 표기**: 자연 베이스라도 고도 가공 추출물일 가능성. 라벨 표기로 구분 불가 (Bobby Approved도 같은 한계)

**제안되는 룰 보완 (Case 1과 별개)**:
7. (검토) 영양 임계 결합 룰 — 100g당 나트륨 1,500mg 초과 → `warn` 또는 별도 nutrition flag
8. (검토) 카테고리 `refined_salt_or_sugar`: "정제염", "정백당", "정제설탕" 단순 표기 잡기 — false positive 위험 큼 (거의 모든 한국 식품에 들어감) → 일반화하지 말고 임계량 결합 권장

### Case 4: Golden Field Rice Bran Oil (2026-05-21) — clean-eating 회색지대 🟡

| 항목 | 값 |
|---|---|
| 바코드 | `8851103220480` |
| 브랜드 | Golden Field (AGRO WELLNESS, 태국) |
| 용량 | 1L |
| 기대 verdict | `okay` |
| **계산 verdict** | `okay` ✅ (일치) |
| 매칭된 bad | (없음) |
| 매칭된 good | (없음) |
| 원재료 | **현미유 100%** (단일 원재료) |
| 영양 참고 | Gamma Oryzanol 10,000mg/L, Vitamin E 11.8mg α-TE/100mL (RDI 107%) |

**관찰**:
- 100% 단일 원재료 — clean eating 원칙 "라벨 단순성" 충족
- 비타민E 풍부 + 감마 오리자놀 (자연 항산화 성분) 강조
- Smoke point 240°C — 튀김에 적합
- 우리 룰 매칭 없음 → `okay`

**clean-eating 진영 내 평가 분기**:
- **중립/우호 진영** (Bobby Approved 등 일부): 단일 원재료 + 자연 영양소 강조 → "neutral" 또는 "okay"
- **엄격한 seed oil 회피 진영**: 모든 정제 식물유(rice bran 포함)는 산업적 hexane 추출 우려 → 회피 권장
- 우리 룰 v1.0.0 은 **중립/우호 진영**을 따라 매칭 없음

**룰 보완 검토 (선택)**:
- **A안 (현 유지)**: rice bran oil 은 우리 룰의 seed_oil 카테고리 밖에 두고 okay
- **B안 (엄격)**: `seed_oil` 카테고리에 "현미유 (정제)", "rice bran oil" alias 추가
  - false positive 위험: 단일 원재료라도 not_okay 라고 표시되어 사용자 혼란
  - "정제" / "압착" 등 추출 방식에 따라 분기하기는 라벨 정보 부족
- **C안 (good 카테고리 분리)**: 추가 `oil_neutral` 카테고리 — okay 이지만 칩으로 "단일 원재료 식물유" 표시

**의사결정 필요**: 시드 더 모이고 사용자 인터뷰 후 진영 결정.

### Case 5: 청정원 곡물100 (이소말토 올리고당) (2026-05-21) — clean-eating 회색지대 🟡

| 항목 | 값 |
|---|---|
| 바코드 | `8801052727523` |
| 브랜드 | 청정원 (대상) |
| 용량 | 1.2 kg |
| 기대 verdict | `okay` |
| **계산 verdict** | `okay` ✅ (일치) |
| 매칭된 bad | (없음) |
| 매칭된 good | (없음) |
| 원재료 | 옥수수전분 99.5%, 쌀 0.5% / 이소말토올리고당 20g 이상 |
| 영양 참고 | 100g당 310 kcal, 당류 44g (RDI 44%) |

**관찰**:
- "곡물 100", "프리바이오틱스" 마케팅 — 자연스럽고 깔끔한 이미지
- 실체: **옥수수전분을 효소 분해한 정제 시럽** (이소말토올리고당 = Isomaltooligosaccharide)
- HFCS(고과당옥수수시럽)와는 다르지만, 옥수수 베이스 가공 시럽이라는 공통점
- 우리 룰 v1.0.0 의 `hfcs` 카테고리는 "고과당옥수수시럽" / "콘시럽" / "corn syrup" 만 매칭 → 옥수수전분/올리고당은 패스

**clean-eating 진영 내 평가 분기**:
- **우호 진영** (마케팅 친화): 자연 곡물 추출 + 프리바이오틱스 + HFCS 아님 → `okay`
- **엄격 진영**: 고도 정제 시럽 + 옥수수 전분 베이스 + 당류 44g/100g → 자연 감미료(꿀/메이플) 권장
- 우리 룰 v1.0.0 은 **우호 진영** 결과

**룰 보완 검토**:
- **A안 (현 유지)**: 옥수수 올리고당은 우리 룰 매칭 밖, `okay` 유지
- **B안 (확장)**: `processed_corn_syrup` 카테고리 신설 — "옥수수 시럽", "이소말토올리고당", "결정과당" 등
  - false positive 위험: 자연 감미료를 강조하는 제품도 잡을 수 있음
- **C안 (good 카테고리)**: 자연 감미료(`honey`, `maple_syrup`, `date`)는 good 으로 잡되, 옥수수 올리고당은 매칭 외 (현재 상태와 동일하지만 사용자에게 chip 으로 표시)

**의사결정 필요**: 사용자 인터뷰 — "옥수수 올리고당 = 자연 단맛 vs 가공 단맛" 어느 쪽으로 인식하는지.

| 항목 | 값 |
|---|---|
| 바코드 | `8801045312316` |
| 브랜드 | 오뚜기 |
| 용량 | 200g (1인분, 170 kcal) |
| 기대 verdict | `okay` |
| **계산 verdict** | `okay` ✅ (일치) |
| 매칭된 bad | (없음) |
| 매칭된 good | (없음) |
| 영양 참고 | 나트륨 950mg/200g (RDI 48%) |

**라벨 원문**:
> 정제수, 감자(미국산: 감자, 산도조절제), 양파퓨레(중국산), 쇠고기(뉴질랜드산), **혼합과일소스**, **변성전분**, 쇠고기브이용, 설탕, **식물성유지**, 카레분, 유크림, 토마토페이스트, 쇠고기다시, 조미양념2, 마늘, 정제소금, 양파맛분말, 후추분, 고춧가루, 강황추출액, **덱스트린**

**관찰**:
- 명백한 합성 첨가물(인공감미료/색소/HFCS/씨앗유 etc) **0건** → 룰 v1.0.0은 `okay`
- 그러나 라벨 곳곳에 **clean-eating 회색지대 토큰**이 다수 분포:
  - `변성전분` (modified starch, E1422 등 가능성)
  - `식물성유지` (palm/seed oil 구분 없는 generic 표기)
  - `덱스트린` (말토덱스트린의 짧은 사촌)
  - `혼합과일소스` (HFCS·과당시럽 가능성, 라벨 불투명)
  - `쇠고기브이용` (bouillon, MSG/효모추출물 가능성)
  - `조미양념2` (코드형 표기, 구성 비공개)
- Bobby Approved 일반적 기준에서는 "회색 또는 회피"에 가까운 라벨

**매칭 안 된 이유**:
- "변성전분" → 룰 미등록 (E1422/E1450 등 구체 표기 없음)
- "식물성유지" → seed_oil 카테고리는 콩기름/카놀라유/옥수수유 등 **구체** alias만 매칭

**제안되는 룰 보완 (Case 1~5 누적과 함께 검토)**:

## 2. 룰 보완 시 고려사항

### false positive 위험
- "향료" 만으로 매칭하면 자연 추출 향료(예: 천연 레몬유 추출물)도 잡힐 수 있음
- "밀가루" 매칭하면 통밀빵·홈베이킹 같은 케이스도 not_okay 됨
- 사용자 신뢰 손실 위험 — "이게 왜 안 좋다는 거지?" 의문

### 보완 시 룰 버전 정책
- 작은 alias 추가(예: "바닐린" 등) → patch 버전 (v1.0.1)
- 새 카테고리 신설(예: `palm_oil_processed`) → minor 버전 (v1.1.0)
- 판정 알고리즘 변경(예: 점수 기반) → major 버전 (v2.0.0)

### 룰 변경 후 작업
1. `packages/happycart_rules/lib/src/bad_ingredients.dart` 수정
2. `verdict_test.dart` / `headline_test.dart` 회귀 테스트 추가
3. `ruleVersion` bump (`verdict.dart`)
4. 시드 fixture 의 `expected_verdict` 재검토
5. `dart run tool/compute_verdicts.dart > supabase/migrations/0005_seed_products.sql` 재실행
6. 마이그레이션 dev/staging/prod 순서로 적용
